# 영상 생성 오류 기록

> 개발 과정에서 발생한 오류, 시도한 해결책, 최종 결론을 정리한 문서입니다.

---

## 오류들

---

### 1. 나노바나나2 — 민감 콘텐츠 거부

**무슨 일이었나**
이미지 생성 프롬프트에 인물의 나이대, 성별, 민족, 피부색 묘사가 포함되어 있었습니다.
Nano Banana 2 모델이 이를 민감 콘텐츠로 판단해 생성을 거부했습니다.

**해결**
Step 3 프롬프트 규칙에서 나이/성별/민족/피부색 관련 묘사를 전부 제거했습니다.
인물 묘사는 의상, 헤어스타일, 소품, 포즈 중심으로만 작성하도록 변경했습니다.

---

### 2. GPT Image 2 PNG → Veo 500

**무슨 일이었나**
인물 일관성을 높이기 위해 GPT Image 2 모델로 이미지를 생성했는데,
이 모델이 PNG 포맷으로 출력합니다. Veo에 PNG 이미지 URL을 전달했더니 500 오류가 발생했습니다.

**시도**
PNG를 JPG로 변환한 뒤 재업로드해서 Veo에 전달해봤지만 여전히 실패했습니다.

**결론**
GPT Image 2를 포기하고 JPG를 출력하는 Nano Banana 2로 복귀했습니다.

---

### 3. Veo — 오디오 필터 400 오류

**무슨 일이었나**
Veo 3.1은 영상을 만들 때 자동으로 오디오(배경음, 나레이션)도 생성합니다.
이때 Google의 오디오 안전 필터가 프롬프트 내용을 검토하는데, 두 가지가 문제가 됐습니다.

- 브랜드명이 따옴표 안에 들어있을 때 (예: "WADIZ CLASS")
- 프롬프트 안에 한국어 나레이션 대본이 포함됐을 때
- `Color: natural skin tones, deep black` 처럼 피부색/인종을 연상시키는 색감 묘사

**해결**
영상 프롬프트를 추출할 때 다음을 자동으로 제거하도록 코드를 수정했습니다:
- 따옴표로 감싼 텍스트 (브랜드명 등)
- 한국어 나레이션 블록
- `Color: ...` 블록 전체

---

### 4. 영상이 720p로 출력되는 문제

**무슨 일이었나**
로그를 확인하니 영상이 1080p가 아닌 720p로 생성되고 있었습니다.
kie.ai Veo API의 기본값이 720p이기 때문이었습니다.

**해결**
영상 생성 요청 시 `resolution: "1080p"` 를 명시적으로 추가했습니다.

---

### 5. `/api/prompts` 500 오류 — BOM 문제

**무슨 일이었나**
Step 2 프롬프트 JSON 파일을 PowerShell로 수정·저장했는데,
PowerShell의 기본 저장 방식이 UTF-8 with BOM이라 파일 앞에 보이지 않는 문자가 붙습니다.
Node.js의 `JSON.parse()`가 BOM이 붙은 파일을 읽으면 파싱 오류가 납니다.

**해결**
JSON 파일 저장 시 반드시 UTF-8 without BOM 방식으로 저장해야 합니다.
PowerShell에서는 `[System.IO.File]::WriteAllText(path, content, New-Object System.Text.UTF8Encoding $false)` 를 사용합니다.

---

### 6. 이미지 URL 경로 문제 (`/images/` vs `/na2/`)

**무슨 일이었나**
Nano Banana 2로 이미지를 생성할 때 출력 URL이
`tempfile.aiquickdraw.com/na2/...` 로 나올 때도 있고
`tempfile.aiquickdraw.com/images/...` 로 나올 때도 있었습니다.
`/images/` 경로일 때 Veo 500 오류가 자주 발생해서 원인으로 의심했습니다.

**시도**
`/na2/` 가 아닌 URL이 나오면 자동으로 이미지를 재생성하는 로직을 추가했습니다.

**결론**
kie.ai 공식 문서 확인 결과, Veo는 JPG/PNG/WebP 모두 지원합니다.
URL 경로가 원인이 아니라 Veo 서버 자체의 간헐적 불안정이 원인이었습니다.
재생성 로직은 불필요한 크레딧 낭비였기 때문에 제거했습니다.

## ⚠️ 아직 완전히 해결되지 않은 문제들

---

### 1. Veo 서버 간헐적 500 오류 (핵심 미해결)

**상황**
같은 이미지, 같은 프롬프트로 요청해도 어떤 때는 성공하고 어떤 때는 500 오류가 납니다.
오류 메시지: `"Internal Error, Please try again later."`

**원인**
Veo 3.1은 아직 Google의 프리뷰/베타 서비스입니다.
kie.ai → Google Veo 서버 사이의 트래픽이 몰리거나 GPU 자원이 부족하면 간헐적으로 실패합니다.
코드나 이미지 형식과는 무관한 서버 측 문제입니다.

**현재 대응**
1. `veo3_fast` 로 1차 시도
2. 실패하면 10초 대기 후 `veo3_fast` 로 재시도
3. 재시도도 실패하면 `veo3_lite` 모델로 전환하여 시도

**한계**
Veo 서버가 지속적으로 불안정할 경우 3번 모두 실패할 수 있습니다.
근본적인 해결책은 없고, Google/kie.ai 서버 상태에 의존하는 문제입니다.

---

### 2. `FIRST_AND_LAST_FRAMES_2_VIDEO` 불안정

**상황**
생성한 이미지 2장을 영상의 첫 프레임과 마지막 프레임으로 고정하는 기능입니다.
이 방식이 `REFERENCE_2_VIDEO`(참고용) 보다 훨씬 높은 일관성을 제공하지만,
오류 발생 빈도도 더 높습니다.

**원인 추정**
`FIRST_AND_LAST_FRAMES_2_VIDEO`는 이미지를 실제 프레임으로 써야 하므로
Veo 서버가 더 많은 리소스를 사용합니다. 서버 불안정 시 더 쉽게 실패합니다.

**현재 대응**
위의 재시도 + veo3_lite 폴백으로 대응 중입니다.

---

### 3. Vercel 배포 환경에서 모델 422 오류

**상황**
Vercel에 배포 후 이미지 생성 시 422 오류가 발생했습니다.
`"The model name you specified is not supported."`

**원인**
환경변수를 Vercel 대시보드에서 수정한 뒤 **재배포를 하지 않으면** 기존 배포에는 새 값이 적용되지 않습니다.

**해결 방법**
환경변수 수정 후 반드시 Vercel에서 **Redeploy** 를 실행해야 합니다.

## 💡 전체 구조 요약

```
원본 이미지 업로드
    ↓
Gemini (Step 2) — 스토리보드 생성
    ↓
Gemini (Step 3) — 이미지 프롬프트 생성 (원본 이미지 분석 기반)
    ↓
Nano Banana 2 — 씬 이미지 2장 생성 (원본 이미지를 시각 레퍼런스로 활용)
    ↓
Veo 3.1 Fast — 이미지 2장을 첫/마지막 프레임으로 영상 생성
    ↓ 실패 시
Veo 3.1 Fast 재시도 → 실패 시 → Veo 3.1 Lite
```
